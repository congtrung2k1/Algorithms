胖胖兮的書架

Time Limit: 5000 ms
Memory Limit: 262144 KB

Description

胖胖兮很喜歡看書，自從他玩過了 Nim 遊戲以後，他和湯湯決定每一個月都要輪流看若干本書， 書本的數量不能超過另外一個人前一個月看書數量的 k 倍。先沒有書看得人就輸了！ 幾個月下來，胖胖兮和湯湯每個月看書的本數已經超過 long long 可以儲存的範圍了！

由於書籍眾多，把書從書架拿下來、放回去總是擺不整齊。 經過了整理一陣以後，胖胖兮想要知道書架上最長的「差不多」排好片段。

所謂的「差不多」排好片段，就是對於一段連續放置的書本， 最左邊的書是這段書本高度之中最矮的，最右邊的書是這段書本之中最高的。

Input Format

第一行有一個正整數 T (1≤T≤100)，代表有幾組測試資料。

每筆測試資料的第一行有一個正整數 n (1≤n≤100000)。 第二行包含 n 個以空白隔開的正整數 a1,a2,⋯,an，代表由左至右每一本書的高度。 為了胖胖兮方便起見，避免拿錯書，每一本書的高度都不相同，並且在 1 到 n 之間。

Output Format

每組測試資料請輸出一行，表示最長的「差不多」排好片段有多長。

Sample Input 

3
5
1 2 3 4 5
5
5 4 3 2 1
5
3 1 2 5 4

Sample Output 

5
1
3
